Java集合

集合框架

概念

​ 对象的容器，实现了对对象常用的操作，类似数组的功能

和数组的区别

1. 数组长度固定，集合长度不固定
1. 数字可以存储基本类型和引用类型，集合只能存储引用类型，（通过装箱）

泛型

Jdk1.5的新特性，其本质是参数化类型，就是把类型当作参数传递

常见形式有 泛型类、泛型接口、泛型方法

语法

​ <T,…> T称为类型占位符 ，表示一种引用类型

好处

​ 提高代码重用性

​ 防止类型转换异常，提高安全性

泛型类

类名后边直接加上 T作为引用类型

创建对象

​ MyGeneric myGeneric = new MyGeneric<>();

泛型集合

概念

​ 参数化类型、类型安全的集合，强制集合元素的类型必须一致

特点

​ 编译时即可检查，而非运行时抛出异常

​ 访问时，不必类型转换 （装箱）

​ 不用泛型之间引用不能相互赋值，泛型不存在多态

在创建集合时 指定其引用类型

在后续操作时 只能使用此类型

如果不指定 则默认Object类型

特点：无序、无下标、元素不可重复

方法：全部继承自Collection中的方法

创建集合

Set set = new HashSet<>();

Set实现类

HashSet 重点

​ 基于HashCode计算元素的存放位置

​ 当存入元素的哈希码相同时，会调用equals进行确认，如果结果为true，则拒绝后者存入

重写hashCode 和 equals方法

直接自动重写

存储过程：

  • 根据hashcode计算保存的位置，如位置为空，则直接保存 否则执行下一步
  • 再指向equals方法，结果为true则认为重复，否则形成链表

TreeSet

​ 基于排序顺序实现元素不重复

​ 实现了SortedSet接口，对集合元素自动排序

​ 元素对象的类型必须实现Comparable接口，指定排序规则

​ 通过CompareTo方法确定是否为重复元素

Comparato实现定制比较 （比较器）

Map集合

Map接口特点：

  1. 用于存储任意键值对（Key-Value）
  2. 键：无序、无下标、不允许重复（唯一）
  3. 值：无序、无下标、允许重复

Map父接口

特点：

存储一对数据，无序、无下标、键不可重复、值可重复

方法：

​ Object get (Object key)// 根据键获取对应的值

​ Set //返回所有key

​ Collection values()//返回包含所有值的Collection集合

​ Set<Map.Entry<K,V>> //键值匹配的Set集合
