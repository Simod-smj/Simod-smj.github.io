Java常用类

内部类

  1. 成员内部类
  2. 静态内部类
  3. 局部内部类
  4. 匿名内部类

概念

​ 在一个类的内部再定义一个完整的类

​ 通常是有依赖关系的

​ 例如 身体类的内部包含头类 手类

特点

  1. 编译之后可生成单独的字节码文件
  2. 内部类可直接访问外部类的私有属性

成员内部类

​ 在类的内部定义，与属性、方法同级别的类

​ 创建内部类对象时，必须依赖外部类对象

​ 当内部类属性和外部类属性名字相同时 会优先访问内部类

​ 访问外部类 为Outer.this.name

静态内部类

成员内部类前加上static关键字 就是静态内部类

static只能在内部类加 外部类不能加

不依赖外部类对象，可以直接创建 或通过类名访问，可声明静态成员

访问外部类属性 需要先实例化外部类

实例化

局部内部类

定义在外部类方法中 ，作用范围和创建对象范围仅限于当前方法

可以直接访问外部类属性

只能在当前方法中使用

​

匿名内部类

没有类名的局部内部类

必须继承一个父类 或者实现一个接口

一般为只使用一次

Object类

超类、基类，所有类的直接或间接父类 老祖宗

​ 常用类

getClass()方法

返回引用中存储的实际对象类型

通常用于判断两个引用中实际存储对象类型是否一致

hashCode()方法

返回该对象的哈希码值

哈希值根据对象的地址或自负串，或数字使用 hash算法 计算出来的int类型的数值

一般情况下相同对象返回相同哈希码 可判断两个对象是否为同一个

toString()方法

返回该对象的字符串表示

可根据需求重写该方法

equals()方法

默认实现为 (this == obj)

比较两个对象的地址是否 相同

可进行重写 比较两个对象内容相同

finalize()方法

当对象被判定为垃圾对象时，由jvm自动调用该方法，用以标记垃圾对象，进入回收队列

垃圾对象：没有有效引用指向此对象时 为垃圾对象

垃圾回收：由GC销毁垃圾对象，释放数据存储空间

自动回收机制：jvm内存耗尽，一次性回收所有垃圾对象

手动回收机制：使用System.gc(); 通知jvm执行垃圾回收

String类

字符串是常量 创建之后不可改变

字符串的字面值存储在字符串池中 可以共享

同时地址发生变化

当 String name2 = “zhangsan”；时

会在字符串池中 寻找有无zhangsan

找到后将name2 也指向zhangsan。此时 name与name2地址相同。即为共享

字符串的另一种创建方式

创建了两个 对象。在堆 跟字符串池中分别创建了一个

![image-20220308171959654](/Users/smj/Library/Application Support/typora-user-images/image-20220308171959654.png)

若此时再创建一个 “java”

![image-20220308173212952](/Users/smj/Library/Application Support/typora-user-images/image-20220308173212952.png)

明显 字符串池中只有一个java。但是在堆中创建了两个对象，所以栈中 str与str2的地址不同

所以，比较字符串是否相等 不能用 str==str2 结果为false ==比较的是地址

​ 要用str.equals(str2) 结果为true equals比较的是内容
